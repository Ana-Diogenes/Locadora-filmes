Projeto Final
Regras Gerais
1. O projeto será desenvolvido em até 3 pessoas.
2. O trabalho final consiste em:
A implementação de um programa funcional.
Uma apresentação em sala de aula explicando o projeto e demonstrando seu funcionamento.
3. Todos os conteúdos vistos na disciplina devem ser aplicados no projeto, incluindo:
Manipulação de arquivos
Uso de bibliotecas
Estruturas de controle e repetição
Organização do código
4. Todo o código deve estar versionado no Git.
5. Plágio não será tolerado. Qualquer uso de código de terceiros sem referência resultará em nota zero.
É proibido o uso do ChatGPT ou qualquer IA para criar o código do projeto. O uso será penalizado com nota zero.

Pré-Requisitos
⚠ Todos os membros do grupo devem conhecer todas as partes do código!
Cada aluno será entrevistado individualmente e deve ser capaz de explicar qualquer parte do projeto.
O projeto será acompanhado em sala de aula e a participação nas atividades contará para a nota .

Requisitos do Programa
O programa desenvolvido deve:
Implementar um problema real, seguindo um dos temas sugeridos abaixo.
Armazenar dados temporariamente, permitindo:
Inclusão
Exclusão
Consulta
Alteração
Gerenciar pelo menos 50 itens armazenados (na apresentação, poderá ter apenas 5).

Requisitos da Apresentação
A apresentação deve ser clara e organizada, contendo:
✔ Explicação do problema resolvido pelo algoritmo.
✔ Demonstração do código e principais trechos.
✔ Execução do programa para mostrar seu funcionamento.
✔ Tempo máximo de 20 minutos.
Critérios Avaliados:
Organização dos slides
Clareza e segurança na apresentação
Postura dos integrantes
Demonstração correta do código em funcionamento
